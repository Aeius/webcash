## How to use git?
1. Branch Strategy
 - 개발: JH/feature/login
 - 핫픽스: JH/hotfix/login
 - 수정: JH/modify/login
 - 담당자/작업내역/기능
 - 작업마다 브랜치생성해야하고, 작업끝나면 삭제
2. Commit rules
 - 개발완료: feat login #issueNum
3. issues
 - 기능단위 todo List 겸으로 issue 발행하기 (기획서)
4. Pull Requests
 - main에 merge 및 push 할 때 PR올리기
